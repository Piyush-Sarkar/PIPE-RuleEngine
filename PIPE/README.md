# PIPE - Real-time Log Analysis & Threat Detection System

A real-time security event monitoring and analysis system that captures website logs, detects attacks in real-time, and generates threat intelligence reports.

## 📋 Overview

PIPE is a three-component system that:

1. **Generates logs** from web application activity
2. **Transmits logs** to the SIEM engine every 20 seconds
3. **Analyzes logs** for security threats in real-time
4. **Generates reports** with attack patterns and suspicious activities

---

## 🏗️ Architecture

```
app.py (Port 5000)
    ↓ (writes logs every few seconds)
LOG_FILE.log
    ↓ (sender.py reads every 20s)
sender.py (Port 5000)
    ↓ (sends NEW logs only every 20s)
siem_rule_engine.py (Port 5001)
    ↓ (receives & appends)
RECEIVE_LOG.log
    ↓ (processes every 20s)
raw_logs.xlsx ← threat_logs.xlsx
(Excel reports with timestamps, IPs, attack types)
```

### Component Details

| Component                     | Port | Role                                                      |
| ----------------------------- | ---- | --------------------------------------------------------- |
| **app.py**              | 5000 | Web application that generates LOG_FILE.log               |
| **sender.py**           | 5000 | Reads new logs from LOG_FILE.log every 20s, sends to SIEM |
| **siem_rule_engine.py** | 5001 | Receives logs, analyzes threats, generates Excel reports  |

## 📦 Running the System

### Terminal 1: Start Web Application (generates logs)

```bash
cd sender
set COLUMN=9999
python app.py > LOG_FILE.log 2>&1
```

- Runs Flask app on `http://192.168.29.19:5000`
- Writes logs to `LOG_FILE.log`

### Terminal 2: Start SIEM Engine (analysis service)

```bash
cd receiver
python siem_rule_engine.py
```

- Runs on `http://127.0.0.1:5001`
- Listens for new logs at `/process-logs` endpoint
- Processes every 20 seconds
- Generates `raw_logs.xlsx` and `threat_logs.xlsx`

### Terminal 3: Start Log Sender (transmission service)

```bash
cd sender
python sender.py
```

- Runs on `http://127.0.0.1:5000`
- Reads `LOG_FILE.log` every 20 seconds
- Sends **NEW logs only** to SIEM engine
- Prints: `📤 Sent new logs to SIEM (X lines)`

## 📊 Output Files

### 1. **RECEIVE_LOG.log** (Receiver folder)

- Raw log file received from sender
- Grows incrementally with new logs only
- Used for input to threat analysis
- **Example line format:**
  ```
  127.0.0.1 - - [27/Apr/2026 10:30:45] "GET /login?id=1' or '1'='1 HTTP/1.1" 401
  ```

### 2. **raw_logs.xlsx** (Auto-generated)

- **Columns:** IP, Date, Time, Method, URL, Status Code
- Parsed logs from RECEIVE_LOG.log
- Sorted chronologically (earliest to latest)
- Updated every 20 seconds

### 3. **threat_logs.xlsx** (Auto-generated)

- **Columns:** IP, Attack, Time, Attack Count
- Only shows detected threats
- Threat types:
  - SQL Injection
  - XSS (Cross-Site Scripting)
  - Session Hijacking
  - Cookie Stealing
  - Keylogging
  - Data Exfiltration
  - Credential Harvesting
  - Directory Traversal
  - LFI (Local File Inclusion)
  - RFI (Remote File Inclusion)
  - DoS (Denial of Service)
  - Brute Force
- Sorted by time (first occurrence to last)
- Updated every 20 seconds

## 🔍 Real-Time Flow

1. **Every few seconds:**

   - `app.py` writes new HTTP request logs to `LOG_FILE.log`
2. **Every 20 seconds:**

   - `sender.py` reads only NEW content from `LOG_FILE.log` (tracks position)
   - Sends new logs to `siem_rule_engine.py` at `http://127.0.0.1:5001/process-logs`
   - Prints: `📤 Sent new logs to SIEM (X lines), Response: 200`
3. **Upon receiving:**

   - `siem_rule_engine.py` appends logs to `RECEIVE_LOG.log`
   - Prints: `✅ SIEM received X new log lines`
4. **Every 20 seconds (analysis):**

   - Reads RECEIVE_LOG.log (tracks position to process only new logs)
   - Converts to `raw_logs.xlsx` (parsed & sorted by timestamp)
   - Analyzes for attacks
   - Generates `threat_logs.xlsx` (threats only, sorted by time)
   - Prints: `⏰ [SIEM] Processed X new logs at YYYY-MM-DD HH:MM:SS`
   - Prints: `⚠️ Threat summary saved → threat_logs.xlsx`

## 🔧 Configuration

Edit `sender.py`:

```python
POLL_INTERVAL_SECONDS = 20  # Change to desired seconds
```

### Change Analysis Interval

Edit `siem_rule_engine.py`:

```python
PROCESS_INTERVAL = 20  # Change to desired seconds
```

### Change SIEM Port

Edit `siem_rule_engine.py`:

```python
uvicorn.run(app, host="127.0.0.1", port=5001)  # Change port here
```

## 📝 Log Format

Expected log format (Apache/Nginx style):

```
127.0.0.1 - - [27/Apr/2026 10:30:45] "GET /path?param=value HTTP/1.1" 200
```

Pattern breakdown:

- `IP_ADDRESS`
- `[DATE TIME]`
- `METHOD URL HTTP/VERSION`
- `STATUS_CODE`

## 🐛 Troubleshooting

### sender.py fails to connect to SIEM

- Ensure `siem_rule_engine.py` is running on port 5001
- Check firewall settings

### No logs appearing in RECEIVE_LOG.log

- Verify `LOG_FILE.log` has new content
- Check sender.py console for error messages
- Ensure `/process-logs` endpoint is reachable

### Excel files not updating

- Check disk space
- Verify file permissions in receiver folder
- Ensure pandas and openpyxl are installed

### Timestamps appearing as NaT (Not a Time)

- Verify log format matches pattern
- Check date/time format in logs
